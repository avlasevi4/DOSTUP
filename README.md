# Реестр дел — обеспечение доступа

Небольшой сайт для отслеживания статуса 18 дел + судебного производства.
Двое людей (вы и коллега) видят одни и те же данные и могут их редактировать —
изменения появляются друг у друга мгновенно.

Технически: страница — статика на GitHub Pages, данные — в Firebase Firestore
(бесплатный тариф с запасом на такой объём). GitHub Pages сам по себе не умеет
сохранять данные с форм — поэтому нужен именно этот тандем.

---

## Шаг 1. Разместить файлы на GitHub Pages

1. Зайдите на github.com → создайте новый репозиторий (можно приватный).
2. Загрузите туда все файлы из этой папки: `index.html`, `styles.css`, `app.js`,
   `seed-data.js`, `firebase-config.js` (через "Add file → Upload files" в браузере
   — командная строка не нужна).
3. Repository → Settings → Pages → Source: выберите ветку `main`, папку `/ (root)`
   → Save.
4. Через 1–2 минуты сайт появится по адресу вида
   `https://ваш-логин.github.io/название-репозитория/`.

Пока не сделан Шаг 2, сайт откроется, но покажет сообщение "Firebase не настроен" —
это ожидаемо.

---

## Шаг 2. Создать бесплатный проект Firebase

1. Зайдите на https://console.firebase.google.com (нужен любой Google-аккаунт).
2. "Добавить проект" → введите название (например, `gmi-access-tracker`) → далее,
   Google Analytics можно отключить — не нужен → "Создать проект".
3. В боковом меню слева: значок `</>` ("Добавить приложение → Web").
   Введите любое название (например, `tracker`), "Также настроить Firebase Hosting"
   — **не отмечать** (нам не нужно, у нас уже GitHub Pages).
4. Появится блок кода с объектом `firebaseConfig` — скопируйте его целиком.
5. Откройте файл `firebase-config.js` (прямо на GitHub — кнопка редактирования,
   значок карандаша) и вставьте свои значения вместо `"ВСТАВЬТЕ_СЮДА"` для
   каждого поля. Сохраните (Commit changes).

---

## Шаг 3. Включить Firestore (базу данных)

1. В консоли Firebase, слева в меню: "Build → Firestore Database" → "Создать базу".
2. Выберите режим **"Начать в тестовом режиме"** (тестовый режим даёт доступ на
   30 дней без ограничений — этого хватит, чтобы всё настроить; правила ниже
   заменят его на постоянные).
3. Выберите регион (любой европейский, например `eur3`) → Включить.
4. После создания базы откройте вкладку **"Правила"** (Rules) и замените
   содержимое на:

   ```
   rules_version = '2';
   service cloud.firestore {
     match /databases/{database}/documents {
       match /{document=**} {
         allow read, write: if true;
       }
     }
   }
   ```

   ⚠️ Это открывает базу для чтения/записи всем, у кого есть ссылка на ваш сайт
   и код доступа (см. Шаг 4). Для двух доверенных людей и не самых секретных
   данных этого достаточно. Если нужна более строгая защита — скажите, можно
   настроить вход по email через Firebase Authentication.

5. Сохранить (Publish).

---

## Шаг 4. Задать свой код доступа

В файле `firebase-config.js` замените строку
```js
export const ACCESS_CODE = "gmi2026";
```
на свой код (любое слово или число) — это простой пароль на входе в сайт,
не настоящая авторизация, но отсекает случайных людей, если ссылка утечёт.
Сообщите этот код коллеге отдельно (не через тот же канал, где ссылка на сайт).

---

## Шаг 5. Проверить

Откройте сайт по ссылке из Шага 1, введите код доступа. При первом открытии
приложение само загрузит в Firestore все 18 текущих дел и данные по судам —
дальше это уже живые данные, редактируемые через интерфейс, а файл
`seed-data.js` больше не используется.

Дайте коллеге ту же ссылку и тот же код доступа — вы оба будете видеть
одни и те же, всегда актуальные данные.

## Как этим пользоваться

- **Реестр дел** — клик по любой строке открывает карточку: можно поменять
  статус, значок, госпошлину, перенести дело в другую группу.
- **Судебное производство** — кнопка "+ Дело в производстве" заводит новую
  карточку; клик по существующей — редактирует.
- **Журнал изменений** — заполняется автоматически при каждом сохранении;
  кнопка "+ Заметка" в шапке — для произвольных записей.
- **Экспорт .md** — выгружает текущий статус в файл, в формате, привычном
  по переписке с Claude — можно вставить в чат для сверки с индексом базы
  знаний проекта.

## Ограничения (честно)

- Код доступа — это не полноценный логин, кто угодно с ссылкой и кодом может
  редактировать любые данные. Для этого масштаба (2 доверенных человека)
  это разумный компромисс между простотой и защитой.
- Нет истории версий по каждому полю (в отличие от индекса в Project
  Knowledge) — только общий журнал текстом. Если нужна такая история —
  можно добавить позже.
- Firestore на бесплатном тарифе — с запасом на такой объём данных и
  количество обновлений (лимит ~50 тыс. чтений/день, вы будете использовать
  в тысячи раз меньше).
